# AGENTS.md

本文件定义 `sosoFlow` 项目的长期开发规则。除非用户明确修改，本文件作为默认执行准则。

## 1. 目标与范围

- 当前项目处于从零初始化阶段。
- 近期目标：先完成可运行的最小骨架，再逐步迭代功能。
- 未经用户明确要求，不新增需求、不改变产品方向。

## 2. 开发原则

- **先读后改**：每次开发前先阅读 `AGENTS.md`、`PROJECT_LOG.md`、`README.md`（若存在）。
- **最小改动**：只做当前任务所需最小修改，不顺手重构无关代码。
- **不重复实现**：已有能力优先复用，避免重复造轮子。
- **一致性优先**：命名、风格、目录、错误处理保持全项目一致。
- **可追踪**：每次完成任务后更新 `PROJECT_LOG.md`（记录做了什么、改了哪些文件、下一步建议）。

## 3. 目录约定（初始化）

在用户未指定技术栈前，保持轻量结构：

- `README.md`：项目说明、启动方式、迭代状态
- `AGENTS.md`：开发规则（本文件）
- `PROJECT_LOG.md`：开发日志与阶段进展
- `src/`：源代码（后续按技术栈细分）
- `tests/`：测试代码（有测试时再创建）

> 说明：若用户后续指定框架/语言，以用户要求为准再细化目录。

## 4. 变更边界

- 不擅自引入大型依赖。
- 不擅自修改项目结构为多包/微服务。
- 不创建与当前任务无关的脚本、配置或文档。
- 不在未确认前进行破坏性操作（删除/重命名核心文件）。

## 5. 任务执行流程

每次任务按以下顺序执行：

1. 明确当前目标与验收标准。
2. 检查现有文件与历史记录，确认无重复工作。
3. 实施最小改动。
4. 做必要验证（至少保证改动可运行或静态可检查）。
5. 更新 `PROJECT_LOG.md` 并给出唯一最高优先下一步。

## 6. 文档语言与风格

- 默认中文说明，必要术语可中英混合。
- 结论优先，简洁表达，避免冗长背景描述。
- 对外输出统一使用：
  - `✅ 项目理解`
  - `📍 当前进度`
  - `🎯 下一步（只选一个最关键的）`
